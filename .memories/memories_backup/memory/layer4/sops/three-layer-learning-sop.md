# 三层学习决策 SOP 🧠

> 来源：Harrison Chase《Continual Learning for AI Agents》
> 目的：遇到任何失败/改进机会时，系统性地决定「该让哪一层学」
> 标签：#continual-learning #sop

---

## 触发条件

**任何时候**你觉得「哪里不对/可以更好」，就触发这个 SOP。

---

## 三层检查清单

遇到问题时，按顺序问自己三个问题：

### Layer 1 → Model？（权重更新）

**该动 Model 层的情况**：
- 问题涉及底层通用能力（推理质量、行为模式、任务能力）
- 当前问题的根因是「模型本身不行」而不是「用法不对」
- 需要的改变应该全局生效、长期有效
- 代价：最高；只在必要时才动

**你在这层能做的**：
- 记录到 MEMORY.md 作为长期观察项
- 建议用户考虑模型升级或微调

---

### Layer 2 → Harness？（执行框架）

**该动 Harness 层的情况**：
- 问题是「工具调用逻辑不对」
- 问题是「流程/工作流设计有缺陷」
- 问题是「异常处理/回退机制缺失」
- 问题是「提示词组织方式不合理」
- 当前失败是因为「框架」而不是「模型能力」或「配置」

**你在这层能做的**：
- 修改 TOOLS.md 中的工具使用说明
- 更新 Skills 的 SKILL.md 中的执行逻辑
- 在 AGENTS.md 中沉淀工作流规范
- 修改 prompt/指令组织方式

---

### Layer 3 → Context？（运行时配置）

**该动 Context 层的情况**：
- 问题是「同类任务之前处理过，但没记住」
- 问题是「知识/偏好/规则没有沉淀下来」
- 问题是「记忆没有跨session保持」
- 问题适合「按用户/按任务」单独维护
- 高频变化、需要快速迭代的内容

**你在这层能做的**：
- 更新 MEMORY.md（长期记忆）
- 更新当天 memory/YYYY-MM-DD.md（日志）
- 更新 Skills 中的配置
- 更新 SOPs
- 写入/更新 TOOLS.md

---

## 决策树

```
遇到问题/改进点
│
├─ 模型能力问题（推理质量差、行为不对）？
│   └─ 是 → Model层：记录到MEMORY.md长期观察
│
├─ 执行框架问题（工具调用、流程、异常处理）？
│   └─ 是 → Harness层：修改SKILL.md/TOOLS.md/AGENTS.md
│
└─ 经验/知识/偏好/记忆没沉淀？
    └─ 是 → Context层：更新memory/SOPs/配置文件
```

---

## 行动模板

遇到值得学习的经验时，用这个格式沉淀：

```markdown
## [事件/问题描述]

**根因分类**：Model / Harness / Context

**Context记录**：
- [具体要记住什么]

**Harness修改**：
- [需要改哪个skill/tool/prompt]

**Model观察**：
- [模型能力层面的问题，观察记录]
```

---

## Context 层是最常用的一层

因为：
- 你没有权限改模型权重（Model）
- 你通常不是OpenClaw框架代码的维护者（Harness）
- **你唯一能高频操作的就是 Context**（memory/sops/skills）

所以：
> 每次失败或学到东西，**第一反应是更新 Context**，只有当同一类问题反复出现且根因在框架时，才考虑 Harness 层。

---

## 执行频率

**每次**产生值得记住的经验时执行。
不要只在失败时执行——正确决策和好结果同样值得沉淀。
