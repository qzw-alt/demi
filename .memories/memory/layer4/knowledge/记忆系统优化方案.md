# 记忆系统优化方案 v2.0

> 问题：每次会话都需要提醒读取记忆文件，记忆更新不及时
> 目标：自动化 + 结构化 + 零提醒

---

## 🎯 核心问题分析

### 当前痛点
1. **MEMORY.md 更新滞后** - 最后更新 2026-02-26，已8天未更新
2. **每日日志分散** - 信息散落在多个日期文件中，难以快速定位
3. **项目记忆隔离不够** - 医疗旅游和龙康劲的信息混杂
4. **缺乏自动加载机制** - 每次都需要手动提醒读取

---

## 💡 解决方案：三层记忆架构

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    HOT MEMORY (热记忆)                       │
│                    自动加载，每次必用                         │
├─────────────────────────────────────────────────────────────┤
│  • AGENTS.md          → 工作规则 (已自动加载)                 │
│  • SOUL.md            → 人格设定 (已自动加载)                 │
│  • USER.md            → 用户信息 (已自动加载)                 │
│  • PROJECT_INDEX.md   → 项目速查表 (新增)                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                   WARM MEMORY (温记忆)                       │
│              按需加载，关键词触发                             │
├─────────────────────────────────────────────────────────────┤
│  • 医疗旅游_项目总览.md    → 医疗旅游完整信息                  │
│  • 龙康劲_项目总览.md      → 龙康劲完整信息                    │
│  • 技术笔记_汇总.md        → 技能使用经验                      │
│  • 待办事项_当前.md        → 当前待办                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                   COLD MEMORY (冷记忆)                       │
│              归档存储，历史查阅                               │
├─────────────────────────────────────────────────────────────┤
│  • memory/2026-XX-XX.md  → 每日原始日志                       │
│  • backup/               → 自动备份                          │
│  • 已完成项目/            → 归档项目                           │
└─────────────────────────────────────────────────────────────┘
```

### Step 3: 更新 AGENTS.md 自动加载规则

在 AGENTS.md 中添加：

```markdown
## Every Session (修改现有规则)

Before doing anything else:

1. Read `SOUL.md` — this is who you are
2. Read `USER.md` — this is who you're helping  
3. Read `PROJECT_INDEX.md` — 项目速查表 (新增！)
4. **If in MAIN SESSION**: Also read `MEMORY.md`

注意：
- 优先使用 PROJECT_INDEX.md 获取项目最新状态
- 需要详细项目信息时，再读取对应的项目总览文件
```

---

### Step 4: 建立记忆更新工作流

**每日工作结束时**：
1. 更新 `memory/YYYY-MM-DD.md` (原始日志)
2. 同步更新 `PROJECT_INDEX.md` (项目状态)
3. 如有重大变更，更新对应的项目总览文件

**每周回顾时**：
1. 整理本周每日日志
2. 更新 MEMORY.md (长期记忆)
3. 归档已完成的项目/任务

### 方案B: 使用 memory-tiering skill

如果 memory-tiering skill 支持，可以：

- 自动归档旧记忆

### 方案C: 会话开始时自动检测

在每次会话开始时：
1. 检查 PROJECT_INDEX.md 是否存在
2. 如果不存在，创建它
3. 如果存在但超过7天未更新，提醒更新
