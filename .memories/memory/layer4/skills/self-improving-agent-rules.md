## 使用规则（已确立）

### 1. 记录范围 - 只记录重要错误

**✅ 记录内容:**
- 工具调用失败（API错误、超时、参数错误）
- 用户明确纠正（"不对，应该用XXX"）
- 工作流低效（发现更好的方法）
- 重复犯同样的错误

**❌ 不记录内容:**
- 临时网络问题
- 用户随意聊天
- 正常的试错过程
- 已记录过的相同错误

### 2. 回顾频率 - 每周一次

**每周回顾内容:**
- 本周记录了哪些错误
- 哪些错误重复出现
- 需要系统性改进的地方
- 更新技能使用规范

**回顾时间:** 每周日晚或周一早

### 3. 分工明确 - 与记忆系统协作

| 系统 | 存储内容 | 使用场景 |
|------|---------|---------|
| **记忆系统** (memory-simple) | 事实、决策、偏好、项目信息 | 查背景、找上下文 |
| **Self-Improving** | 错误、纠正、改进、最佳实践 | 避免重复错误、持续优化 |

**协作方式:**
```
遇到错误 → Self-Improving 记录 → 分析原因 → 生成改进方案
需要决策 → 查询记忆系统 → 获取背景 → 做出决策
每周回顾 → 整合两个系统 → 更新规范 → 持续进化
```
