> 今天安装了 multi-search-engine 技能，并建立了高效的搜索工作流程

### 技能信息
| 项目 | 详情 |
|------|------|
| **技能名称** | multi-search-engine |
| **版本** | v2.0.1 |
| **作者** | gpyangyoujun |
| **安装位置** | `~/.openclaw/workspace/skills/multi-search-engine/` |
| **特点** | 17个搜索引擎（8国内+9国际），无需API Key |

### 搜索策略（已确立为标准工作流程）

```
┌─────────────────────────────────────────────────────┐
│  用户提出搜索需求                                      │
│       ↓                                              │
│  ┌──────────────────┐                                │
│  │ Step 1: Multi Search │ ← 广度搜索，多来源对比       │
│  │ (DuckDuckGo/Google)  │    快速获取概览             │
│  └────────┬─────────┘                                │
│           ↓                                          │
│  ┌──────────────────┐                                │
│  │ Step 2: Tavily     │ ← 精准搜索，深度分析          │
│  │ (需要时启用)       │    提取具体数据               │
│  └────────┬─────────┘                                │
│           ↓                                          │
│  整合信息 → 形成完整答案                               │
└─────────────────────────────────────────────────────┘
```

### 使用场景对比

| 场景 | 推荐工具 | 原因 |
|------|---------|------|
| 一般信息查询 | **Multi Search** | 多引擎覆盖，快速定位 |
| 价格对比 | **Multi Search** | 多来源验证 |
| 新闻/实时信息 | **Multi Search** | 快速获取多平台信息 |
| 具体数据/排名 | **Tavily** | 深度分析，提取精确数字 |
| 竞争对手分析 | **Tavily** | 结构化输出，便于整理 |

### 今天实战案例：医院排名博客

**任务**：创建中国医院排名2026博客文章

**执行过程**：
1. **Multi Search (DuckDuckGo)** → 搜索"中国医院排行榜2025 复旦版"
   - 快速获取：复旦版排名改为等级制、TOP10医院名单
   - 发现权威来源：健康界、知乎、今日头条

2. **Tavily 深度搜索** → 获取具体排名数据
   - 骨科TOP10：积水潭医院 #1 (10.064分)、上海六院 #2 (8.679分)
   - 心血管TOP3：阜外医院、中山医院、安贞医院
   - 肿瘤科TOP5：医科院肿瘤医院 #1 (10.923分)
   - 神经外科TOP3：天坛医院、华山医院、宣武医院

3. **整合输出** → 24,597字节完整HTML文章
   - 包含：综合排名、4个专科排名、城市推荐、成本对比

**效率提升**：
- 以前：多次搜索，信息零散
- 现在：2步完成，信息完整

### 技术细节

**Multi Search Engine 支持的引擎**：
- **国内**：百度、必应中国/国际、360、搜狗、微信、今日头条、集思录
- **国际**：Google、Google HK、DuckDuckGo、Yahoo、Startpage、Brave、Ecosia、Qwant、WolframAlpha

**使用方式**：
```javascript
// 基础搜索
web_fetch({"url": "https://duckduckgo.com/html/?q=搜索关键词"})

// Google 搜索
web_fetch({"url": "https://www.google.com/search?q=搜索关键词"})
```

### 方法论总结

**核心原则**：**先广后深，层层递进**

1. **广度优先**：先用 Multi Search 快速扫描，建立信息地图
2. **精准打击**：再用 Tavily 深入挖掘具体数据
3. **交叉验证**：多来源对比，确保信息准确
4. **及时记录**：将搜索策略固化为工作流

**避坑指南**：
- Google 搜索可能被拦截，优先使用 DuckDuckGo
- Tavily 有额度限制，关键查询再用
- 搜索结果要标注来源，便于追溯
