## 🕳️ 踩坑记录（经验教训）

### 错误1：Write工具参数遗漏
**问题：** 连续3次调用write工具忘记加`file_path`参数

**错误代码：**
```javascript
// ❌ 错误
write({ content: "..." })

// ✅ 正确
write({ content: "...", file_path: "/path/to/file" })
```

**原因：** 惯性思维，以为content是主要参数就忘了路径

**解决方案：**
- 每次调用write先检查两个必需参数
- `file_path`和`content`缺一不可
- 长内容先写简化版，再完善

**影响：** 浪费3次API调用，延迟任务完成

---

### 错误2：路径参数混淆
**问题：** 不确定用`path`还是`file_path`

**发现：** 
- 不同工具参数名不同
- read工具：`file_path`
- write工具：`file_path`
- 有些旧工具可能用`path`

**解决方案：**
- 看工具定义，按规范来
- 出错时检查参数名
- 统一使用`file_path`

---

### 方法论总结

| 场景 | 正确做法 |
|------|---------|
| 写文件 | 必须包含file_path + content |
| 长内容 | 先写框架，再补充细节 |
| 不确定参数 | 看工具定义或试错 |
| 批量操作 | 晚上统一处理 |

**核心原则：先验证，再执行**
