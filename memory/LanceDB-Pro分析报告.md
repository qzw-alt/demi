---

## 🚀 核心能力

### 1. 混合检索 (Hybrid Retrieval)

```
查询 → 向量搜索 (语义相似)
     → BM25全文搜索 (关键词匹配)
     → 融合排序 → 结果
```

**比我们的三层记忆强在哪里**:
- 向量搜索: 理解语义，不只是关键词匹配
- BM25搜索: 精确关键词匹配
- 混合融合: 两者结合，召回率更高

### 2. 多阶段评分管道

| 阶段 | 功能 | 效果 |
|------|------|------|
| **时间衰减** | 新记忆分数更高 | 自动优先近期 |
| **重要性加权** | 重要记忆分数更高 | 关键信息优先 |
| **长度归一化** | 防止长文本垄断 | 公平排序 |
| **MMR多样性** | 相似内容去重 | 结果更多样 |
| **交叉编码器重排序** | AI重新排序 | 更精准 |

### 3. 自适应检索

- **自动跳过**: 问候语、简单确认、表情符号
- **强制检索**: 包含"记得"、"之前"、"上次"等关键词
- **中文优化**: CJK感知阈值

### 4. 噪音过滤

自动过滤低质量内容:
- 拒绝回答 ("我没有信息")
- 元问题 ("你还记得吗")
- 问候语 ("你好"、"嗨")

### 5. 自动捕获 & 自动回忆

- **会话结束**: 自动提取偏好/事实/决策，存储到数据库
- **会话开始**: 自动注入相关记忆到上下文

### 6. 多作用域隔离

- `global` - 全局记忆
- `agent:<id>` - 特定agent记忆
- `project:<id>` - 项目记忆
- `user:<id>` - 用户记忆

## 💡 建议：恢复使用

### 为什么应该恢复？

1. **向量搜索**: 能理解语义，不只是关键词
   ```
   用户: "上次说的医院"
   三层记忆: 需要精确匹配"医院"
   LanceDB: 能理解"医疗机构"、"诊所"等同义词
   ```

2. **自动捕获**: 不用手动记录，自动提取重要信息

3. **智能回忆**: 只在需要时加载相关记忆，不是每次都加载全部

4. **噪音过滤**: 自动过滤无用对话，只保留有价值的信息

### 如何结合使用？

```
三层记忆 (HOT/WARM/COLD):
  - 结构化信息 (项目状态、待办事项)
  - 人工维护，精确控制
  - 每次会话必读的上下文

LanceDB Pro:
  - 非结构化信息 (对话历史、决策记录)
  - 自动捕获，智能检索
  - 按需加载的相关记忆
```

**组合效果**: 
- 三层记忆提供**稳定的框架**
- LanceDB Pro提供**智能的填充**

## 🛠️ 恢复步骤

### Step 1: 从备份恢复

```bash
cd ~/.openclaw/workspace
cp -r backup/skills/memory-lancedb-pro skills/
```

### Step 2: 安装依赖

```bash
cd skills/memory-lancedb-pro
npm install
```

### Step 3: 配置 OpenClaw

编辑 `~/.openclaw/openclaw.json`:

```json
{
  "plugins": {
    "load": {
      "paths": ["skills/memory-lancedb-pro"]
    },
    "entries": {
      "memory-lancedb-pro": {
        "enabled": true,
        "config": {
          "embedding": {
            "provider": "openai",
            "model": "text-embedding-3-small",
            "apiKey": "${OPENAI_API_KEY}"
          },
          "retrieval": {
            "topK": 5,
            "vectorWeight": 0.7,
            "bm25Weight": 0.3
          }
        }
      }
    }
  }
}
```

### Step 4: 测试

```bash
openclaw memory stats
```
