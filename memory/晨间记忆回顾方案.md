# 🌅 晨间记忆回顾方案

> 目标：每天早上自动回顾过往工作和技能，为当天工作做好准备
> 时间：每天早上 06:00 (GMT+8)

---

## 📋 方案概述

### 核心思想
就像人类早上醒来会回忆昨天做了什么、今天有什么计划一样，AI 也需要一个**晨间记忆回顾**的过程，确保：
1. ✅ 掌握所有项目最新状态
2. ✅ 记住已安装的技能和工具
3. ✅ 明确今天的待办事项
4. ✅ 回顾昨天的成果和教训

---

## 🎯 晨间回顾内容

### 1. 核心记忆读取 (必读)
```
🔥 HOT:   memory/hot/HOT_MEMORY.md      → 当前活跃任务
🌡️ WARM:  memory/warm/WARM_MEMORY.md    → 稳定配置和偏好
📋 INDEX: PROJECT_INDEX.md              → 项目速查表
```

### 2. 昨日工作回顾
```
📅 读取: memory/YYYY-MM-DD.md (昨天)
📊 回顾: 昨天完成了什么？
⚠️ 检查: 有什么问题需要跟进？
```

### 3. 今日待办确认
```
🎯 检查: PROJECT_INDEX.md 中的待办列表
📝 创建: memory/YYYY-MM-DD.md (今天)
🔥 更新: HOT_MEMORY.md 活跃任务
```

### 4. 技能和工具清单
```
🛠️ 已安装技能 (快速回顾)
🔑 API Keys 状态检查
📚 常用工作流确认
```

## ⏰ 实施方式

### 方式A: OpenClaw Cron 任务 (推荐)

创建一个每天早上 6:00 运行的 cron 任务：

```yaml
# cron 配置
schedule: "0 22 * * *"  # UTC 22:00 = 中国时间 06:00
task: morning-memory-review
```

**任务内容**：
1. 读取所有核心记忆文件
2. 生成晨间回顾报告
3. 发送给用户 (可选)
4. 更新 HOT_MEMORY.md

### 方式B: 手动触发

用户每天早上发送消息：
```
"晨间回顾" 或 "morning review"
```

AI 自动执行回顾流程。

### 方式C: 首次对话自动触发

如果当天还没有进行过晨间回顾，首次对话时自动执行：
```
[系统检测到今日尚未进行晨间回顾]
🌅 正在执行晨间记忆回顾...
...
✅ 回顾完成，准备好开始工作！
```

## 🚀 立即实施

### Step 1: 创建 Cron 任务

使用 OpenClaw 创建定时任务：

```bash
openclaw cron create \
  --name "晨间记忆回顾" \
  --schedule "0 22 * * *" \
  --command "morning-memory-review" \
  --channel feishu
```

### Step 2: 创建任务脚本

创建一个完整的晨间回顾任务脚本，包含：
- 读取所有记忆文件
- 生成回顾报告
- 发送给用户

### Step 3: 测试运行

手动触发一次，确保流程正常：
```bash
openclaw cron run --name "晨间记忆回顾" --now
```
