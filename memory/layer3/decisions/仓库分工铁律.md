# 铁律规则 - 仓库分工（不可违反）

> 最后更新：2026-03-19
> 教训来源：误删文件事件

---

## 🔴 绝对规则（硬性铁律）

### 1. 仓库分工（永不混淆）

| 仓库 | 用途 | 允许内容 | 禁止内容 |
|------|------|----------|----------|
| **qzw-alt/demi** | 完整备份 | 所有文件（代码、文档、技能、记忆、配置） | 无 |
| **qzw-alt/chinahospitalsguide** | 网站部署 | 仅网站文件（HTML、CSS、JS、图片、配置文件） | 任何非网站文件 |

### 2. 网站文件定义（明确边界）

**允许上传的文件类型：**
- HTML 文件 (*.html)
- CSS 文件 (*.css)
- JavaScript 文件 (*.js)
- 图片文件 (*.jpg, *.png, *.svg, *.gif, *.webp)
- 字体文件 (*.woff, *.woff2, *.ttf)
- 配置文件：CNAME, robots.txt, sitemap.xml, .nojekyll, netlify.toml
- 网站根目录下的必需文件

**禁止上传的文件类型：**
- Markdown 文档 (*.md) - 除非是网站内容必需
- Python 脚本 (*.py)
- Shell 脚本 (*.sh)
- 技能目录 (skills/)
- 记忆目录 (memory/)
- 配置目录 (.agents/, .clawhub/, .learnings/)
- 开发文档、笔记、SOP
- 任何与网站运行无关的文件

### 3. 操作流程（强制执行）

**每次部署前必须检查：**

```bash
# 1. 确认当前仓库
git remote -v

# 2. 如果是 chinahospitalsguide，只添加网站文件
git add *.html *.css *.js *.jpg *.png *.svg

# 3. 禁止添加非网站文件
git add *.md *.py *.sh skills/ memory/ .agents/ .clawhub/ .learnings/
```

### 4. 备份策略（双重保险）

**demi 仓库（完整备份）：**
- 每天自动备份所有文件
- 包含：skills, memory, .agents, 文档, 脚本, 网站文件
- 提交信息："backup: 完整备份 YYYY-MM-DD"

**chinahospitalsguide 仓库（仅网站）：**
- 只从 docs/ 目录或网站专用文件夹复制
- 提交前必须清理非网站文件
- 提交信息："deploy: 网站更新 YYYY-MM-DD"

### 5. 错误预防（强制检查）

**提交前必须运行：**

```bash
# 检查是否有非网站文件被添加
git status | grep -E "\.md$|\.py$|\.sh$|skills/|memory/|\.agents/|\.clawhub/|\.learnings/"

# 如果有输出，立即停止提交
```

**如果误添加，立即撤销：**

```bash
# 撤销特定文件的添加
git reset HEAD <file>

# 或者撤销所有添加，重新选择
git reset HEAD
```

---

## ⚠️ 违规后果

- 网站部署失败
- GitHub Pages 404 错误
- 文件丢失
- 历史记录混乱
- 需要手动恢复（耗时费力）

## ✅ 正确示例

### 备份到 demi 仓库：
```bash
cd ~/.openclaw/workspace
git add -A  # 添加所有文件
git commit -m "backup: 完整备份 2026-03-19"
git push origin master
```

### 部署到 chinahospitalsguide 仓库：
```bash
# 只复制网站文件到干净目录
mkdir -p /tmp/website_deploy
cp *.html /tmp/website_deploy/
cp -r blog/ /tmp/website_deploy/
cp -r stories/ /tmp/website_deploy/
cp CNAME robots.txt sitemap.xml .nojekyll /tmp/website_deploy/

# 在干净目录提交
cd /tmp/website_deploy
git init
git add -A
git commit -m "deploy: 网站更新 2026-03-19"
git push origin master --force
```
