# Agent 持续学习框架 — Harrison Chase (LangChain)

> 来源：Harrison Chase（LangChain创始人）文章《Continual Learning for AI Agents》
> 整理：伟烨分享
> 标签：#continual-learning #agent #架构

---

## 核心观点

传统理解太窄——持续学习不止发生在模型层（微调/RL），**Harness 和 Context 都可以是学习发生的位置**。

---

## 三层学习框架

### Layer 1：Model（模型权重）

**学进去的**：推理模式、任务能力、行为倾向

| 维度 | 特点 |
|------|------|
| 形态 | 权重文件 |
| 成本 | 最高 |
| 速度 | 最慢 |
| 可检查性 | 差 |
| 影响上限 | 最高（全局性、长期性） |
| 更新方式 | 通常按 Agent 级，离线更新 |
| 典型方法 | 微调、强化学习 |

**问题**：
- 成本高、周期长、可检查性差
- **灾难性遗忘**：新数据可能冲掉旧能力
- 难以细化到「按用户/租户」分别更新

---

### Layer 2：Harness（执行框架）

**包在模型外面的所有执行系统**——代码、固定提示词、工具调用逻辑、工作流、异常处理

| 维度 | 特点 |
|------|------|
| 形态 | 代码 |
| 成本 | 中间 |
| 速度 | 中等 |
| 可检查性 | 强（代码可review） |
| 影响上限 | 中等 |
| 更新方式 | 按 Agent 级，离线更新 |

**学习机制**：
1. 跑一批任务 → 产生 Execution Traces
2. 对结果做评估
3. 日志和轨迹存下来
4. Coding Agent 分析 traces → 提出 Harness 代码修改建议

**更新进去的可能是**：提示词组织方式、工具调用逻辑、任务分解策略、异常处理流程、失败回退方式

---

### Layer 3：Context（运行时外部配置）

**最贴近当前很多 Agent 工程实践的一层**

| 维度 | 特点 |
|------|------|
| 形态 | 配置文件（skills、memory、instructions、tools） |
| 成本 | 最轻 |
| 速度 | 最快 |
| 可检查性 | 最强 |
| 影响上限 | 最低（低于模型层） |
| 更新方式 | 可离线，也可运行中实时更新 |
| 粒度 | 可按 Agent / 用户 / 团队 / 租户分别维护 |

**学习方式**：
- 离线：任务结束后分析 traces，批量更新 Context
- 在线：执行过程中直接更新 Memory（边做边学）

**关键区分**：同样是指令/技能/工具，区别在于——是**固定框架**还是**运行时配置**

---

## 三层对比表

| | Model | Harness | Context |
|--|-------|---------|---------|
| **形态** | 权重 | 代码 | 配置文件 |
| **成本** | 最贵 | 中间 | 最轻 |
| **速度** | 最慢 | 中等 | 最快 |
| **可检查性** | 最难 | 强 | 最容易 |
| **影响上限** | 最高 | 中等 | 最低 |
| **粒度** | Agent级 | Agent级 | 可细分到用户/团队/租户 |
| **更新方式** | 离线 | 离线 | 离线+在线 |

---

## 决策框架

| 问题类型 | 应该动哪层 |
|---------|-----------|
| 长期底层通用能力问题 | Model |
| 执行稳定性、工具使用、流程编排问题 | Harness |
| 偏好、记忆、组织、知识、角色差异 | Context |

---

## 企业额外约束：权限边界

Box CEO Aaron Levie 补充：

一旦知识进入 Model 层，就很难再按权限切分，也很难解释「某个输出到底受了哪些内部信息影响」。

→ **企业持续学习更适合落在 Context 层**，因为 Context 天然支持按用户/团队单独维护，更容易和权限体系绑定。

---

## 核心：Execution Traces 是三层共同的学习原料

- Model 层更新需要 traces 作为训练材料
- Harness 层更新需要 traces 定位问题、改代码、改流程
- Context 层更新需要 traces 抽取经验、更新记忆和配置

**Trace 不只是日志/观测数据/调试工具——它是 Agent Learning 的基础设施。**

---

## 关键结论

> Agent 要持续变好，关键不是只换更强的模型，而是**想清楚该让哪一层学**。

- Model 学的是**能力**（代价最高，上限最高）
- Harness 学的是**如何执行**（核心机制：traces + eval + code）
- Context 学的是**经验和配置**（更新最快，力度最细，最适合高频沉淀）
